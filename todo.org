* Review findings (2026-05)

Outcome of the exhaustive review. Tier ordering = bang-for-buck.

** P0 — Bugs / smells worth fixing now

*** DONE Remove dead-code stubs in =src/ui/mod.rs=
=let _ = MarketSide::Buy;= and =let _ = header::CONTENT_TYPE;=
inside =handle_book_ws=. Both imports were genuinely unused — dropped
the stubs and the imports.

*** DONE Fix =/api/info= coupling undercount
Added =World::coupling_count()= which walks the actual couplings
map (every edge stored as A→B + B→A, halved for the unique count).
=/api/info= calls it directly. Removed the gridpool-area indirection
and the inline =/ 2= dedup.

*** DONE MM goes silent on spread collapse
Reordered =MarketMaker::refresh= to compute the new bid/ask
*before* cancelling. When demand+surplus collapse the spread
(bid_price >= ask_price), leaves the previous quotes resting
instead of wiping them — no more zero-MM-quote window for
aggressors to slip through.

*** DONE Trade WS snapshot on connect
=handle_trades_ws= now drains =public_trade_history()= before
attaching to the live broadcaster, same single-lock
snapshot-then-subscribe pattern the book WS uses. Page reloads
fill the trade tape immediately instead of waiting for the next
print.

*** DONE Persist dropdown choices in localStorage
Switched away from relying on browser form-restoration (unreliable
for selects whose options arrive after parse time). =SELECT_PREFS=
table + =rememberSelectChoice= / =rememberedSelectChoice= helpers
back all three dropdowns: chart window, book delivery period,
trades filter. Boot reads localStorage; user changes write through;
=rerenderBook= / =rerenderTrades= drop stale persisted values that
no longer match a live contract.

*** DONE Unify =next_quarter_boundary=
=scenarios.rs= now imports the existing =lisp::next_quarter_boundary=
instead of carrying its own duplicate. One source of truth.

** P1 — Refactors that simplify a lot

*** DONE Extract =fundamentals= helper in =apply_biases=
Introduced an in-scope =fundamentals(area_code, offset) -> Decimal=
closure that captures =curve=, =weather=, =period_hour_for=, and
=day_of_year=. Both loops now read like "compute side-bias /
demand / surplus, write fields". Also folded the MM's redundant
=imbalance= temporary into the =shift= computation.

*** DONE =Order::limit= constructor
Added =Order::limit(area, period, side, price, quantity, currency)=
that returns a LIMIT order with the optional fields left None.
Counterparty MM `build`, the aggressor IOC submission (via struct
update syntax), the slippage regression test, and the world / gridpool
sample fixtures all collapsed onto it. The submit-error test cases that
override one field via =..sample_buy(...)= keep their struct-update form;
no need to break that pattern.

*** DONE Merge =World::submit_order= + =submit_counterparty_order= core
Extracted =admit_and_match(order, taker_gridpool, now)= for the
shared "mint id → resolve mode → match → record fills" core.
=submit_order= keeps the gridpool checks (pool exists, area
allow, AON / valid_until rejection, self-trade prevention) and
the post-match work (OrderDetail + state machine + pool record +
binding + fanout); =submit_counterparty_order= is now a 3-line
wrapper that validates + calls the core.

*** DONE Generic server filter helpers
Four little =bool=-returning helpers — =enum_list_passes=,
=optional_eq_passes=, =area_passes=, =exact_period_passes= — cover
the repeated shapes across the four filter predicates. Each of
=matches_gridpool_trade_filter=, =matches_public_book_filter=,
=matches_public_trade_filter=, =matches_order_filter= collapses
to a single short =&&= chain. Net -41 LOC, easier to read.

*** TODO Split =src/lisp/mod.rs= into a directory
1575 LOC monolith. Suggested split: =config.rs= (struct + boot),
=specs.rs= (spec types), =defuns/= (one file per cluster: MMs,
aggressors, weather, curve, scenarios, metadata), =watch.rs=,
=timers.rs=. Mechanical refactor, zero behavior change.

*** DONE =mk_setter= for aggressor setters
=mk_setter= is now generic over the spec type via an =access=
closure that hands back the spec's =Arc<RwLock<Config>>=. Two
small =mm_cfg= / =ag_cfg= fn pointers cover the two specs; the
aggressor's =set-aggressor-size= + =set-aggressor-side-bias= now
go through the same helper as every MM setter.

*** DONE Split =ui-assets/index.html= into three files
=index.html= (125 LOC), =style.css= (700 LOC), =app.js= (1117
LOC). =index.html= references the other two via =<link>= /
=<script src>=. New =/style.css= and =/app.js= axum routes
serve them with appropriate Content-Type headers.

*** DONE Unify the three UI area lists
Single =ALL_AREAS= array now holds every configured zone with
=code=, =tag=, =group=, =color=. =DE_AREAS= and =CHART_AREAS=
derive via =.filter()=; =ALL_FILTER_AREAS= is an alias for callers
that already use that name. Removing duplicated zone definitions
also means adding a new area only requires touching one place.

** P2 — Architecture observations worth discussing

These are bigger refactors. Each deserves a dedicated design pass
before swinging the hammer.

*** TODO Split =World= into smaller types
2375 LOC owning markets, gridpools, books, couplings, fan-out,
history, indices, ids, suspended flag, AND matcher orchestration.
Natural slices: =MarketState= (physical state), =Fanout=
(broadcasters + history), =Matcher= (free fns / new type).

*** TODO Unify MM and Aggressor under a =Counterparty= trait
Both hold =Arc<RwLock<Config>>=, a seeded RNG, a tick loop, a
fair-value reference, a price/size policy, and submit via
=submit_counterparty_order=. A small trait sharing the spawn
pattern would deduplicate =spawn_mm_task= / =spawn_aggressor_task=.

*** DONE Move duck curve out of =scenarios.rs=
=natural_duck_bias=, =decay_weight=, =lerp= now live in
=sim::demand=. =scenarios.rs= re-exports them so the bias-tick
call sites still work. =effective_ref= stays in =scenarios.rs= —
it's tightly coupled to the bias tick's curve + weather lookup.

*** DONE =Config::with_defaults()= to kill =bin/tradingsim.rs= branching
=Config::new= and =Config::with_defaults()= share a private
=build_empty(filename)= scaffolding fn. The binary always holds
a real =Config= — no more =Option<LispConfig>=, no more
per-section =unwrap_or_else= fallbacks. The "no config.lisp
present" path constructs the defaults-only Config and proceeds
identically.

*** DONE Exponential backoff on WS reconnect
=tradesWsBackoff= / =bookWsBackoff= live in =app.js=. Start at
1s, double on each =onclose= (capped at 30s), reset to 1s on a
successful =onopen=. Both WS reconnect paths use the same
constants.
